- InlineCalendar에서 스크롤에 따라 Header의 표시 연월을 동기화하는 기능이 고장났어. 스크롤 하다보면 다시 오늘 날짜로 되돌아와버려. 아마 전역 상태로 관리하면서 문제가 생긴 것 같은데 확인 부탁해.

## 해결 방법:

1.  **`App.tsx` 수정**:
    *   사용자가 클릭하여 선택한 날짜를 위한 `selectedDate` 상태와 별개로, 캘린더의 현재 보이는 월/연도를 추적하는 새로운 `currentCalendarViewDate` 상태를 도입했습니다.
    *   `handleDateSelect` 및 `handleSelectToday` 함수는 이제 `selectedDate`와 `currentCalendarViewDate`를 모두 업데이트합니다.
    *   `Header` 컴포넌트에 `currentCalendarViewDate` (날짜 객체)를 `currentCalendarViewDate` prop으로 전달하고, `handleCalendarViewChange` 함수를 `Header`에 전달합니다.

2.  **`Header.tsx` 수정**:
    *   `HeaderProps` 인터페이스를 `currentCalendarViewDate` (날짜 객체)와 `onCalendarViewChange` (날짜 콜백)를 반영하도록 업데이트했습니다.
    *   헤더에 표시되는 월/연도 문자열을 포맷팅하기 위해 `currentCalendarViewDate`를 사용합니다.
    *   `onCalendarViewChange`를 `InlineCalendar`에 `onViewChange` prop으로 전달합니다.

3.  **`InlineCalendar.tsx` 수정**:
    *   `InlineCalendarProps` 인터페이스를 `onViewChange` (날짜 콜백)를 사용하도록 업데이트하고 `onMonthYearChange`를 제거했습니다.
    *   내부 `selectedDate` 상태와 해당 동기화 `useEffect`를 제거했습니다. 이제 `selectedDateProp`이 강조 표시를 위한 선택된 날짜의 단일 진실 소스입니다.
    *   `calculateVisibleDates` 함수는 이제 `centerDate` (날짜 객체)와 함께 `onViewChange(centerDate)`를 호출하여 현재 보이는 날짜를 보고합니다.
    *   초기 스크롤을 위한 `useLayoutEffect`는 이제 `selectedDateProp`을 우선적으로 스크롤하고, 그렇지 않으면 `new Date()`로 폴백합니다.

이러한 변경 사항은 헤더에 표시되는 월/연도와 사용자가 선택한 특정 날짜를 올바르게 분리하여, 헤더가 스크롤된 뷰를 정확하게 반영하고 선택된 날짜가 재설정되지 않도록 보장합니다.
