# 작업 내역 (2026-01-07)

## 요청 사항
- `404.html` 리다이렉트 시 발생하는 무한 루프 현상 해결 및 유효하지 않은 경로 처리.

## 수행 내용
1. **무한 루프 방지 로직 (`public/404.html`)**:
   - 이미 쿼리 스트링(`?/`)이 포함된 상태에서 다시 404가 발생하면 리다이렉트를 중단하고 루트(`/`)로 강제 이동시키는 로직을 추가했습니다.
2. **경로 설정 최적화**:
   - `${org}.github.io` 루트 배포 기준에 맞춰 `pathSegmentsToKeep = 0`으로 수정했습니다.
3. **가이드 업데이트**:
   - 루트 도메인 배포 사례를 중심으로 가이드 내용을 간소화했습니다.

## 결과
- 존재하지 않는 경로(예: `/123123`)로 접근 시 무한 루프 없이 루트 페이지로 안전하게 돌아옵니다.
