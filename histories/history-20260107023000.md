# History - 2026-01-07

## 작업 요약
- 모바일 환경에서 프로젝트가 선택되지 않았을 때(또는 없을 때) 사이드바가 자동으로 열리도록 수정했습니다.
- 프로젝트 미선택 시 나타나는 안내 화면을 클릭하면 사이드바가 열리도록 개선했습니다.
- `DailyDetailForm`의 입력 필드가 모바일 화면 밖으로 튀어나가는 레이아웃 오류를 수정했습니다.

## 상세 내용
### 1. `src/App.tsx`
- `selectedProjectId`가 `null`일 때(프로젝트 미선택 상태) `showSidebar` 상태를 `true`로 설정하는 `useEffect` 훅을 추가했습니다.
- 프로젝트가 선택되지 않았을 때 나타나는 안내 문구 영역(`프로젝트를 생성하거나 선택하세요.`)에 `onClick` 핸들러를 추가하여, 해당 영역을 클릭하면 사이드바가 열리도록 개선했습니다.
- 클릭 시 시각적 피드백을 위해 `cursor-pointer` 및 모바일용 `active` 스타일을 추가했습니다.

### 2. `src/components/DailyDetailForm.tsx`
- 입력 필드(`input`)의 클래스에 `min-w-0`을 추가했습니다.
- Flexbox 컨테이너 내부에서 입력 필드가 고유 너비(intrinsic width)를 유지하려다 부모 컨테이너를 넘치는 현상을 방지하여, 작은 화면에서도 레이아웃이 깨지지 않도록 조치했습니다.

### 3. `src/App.tsx` (추가 수정)
- `handleAddProject` 함수에서 프로젝트 생성 후 자동으로 사이드바가 닫히도록 `setShowSidebar(false)`를 추가했습니다. 이로써 사용자가 새 프로젝트를 만들고 바로 작업 화면을 볼 수 있게 되었습니다.
- `handleDateSelect`의 `useCallback` 의존성 배열에서 불필요한 `handleSelectProject`를 제거하여 코드를 정리했습니다.
