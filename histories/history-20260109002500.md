# 작업 히스토리 - 2026-01-09 (4)

## 작업 내용
할 일(Task) 추가 시, 선택 가능한 유효한 목표(Goal)가 없는 경우 불필요한 목표 선택 Bottom sheet를 띄우지 않고 즉시 등록되도록 UX를 개선했습니다.

### 주요 변경 사항
1.  **할 일 추가 로직 개선** (`src/App.tsx`)
    *   `handleAddTask` 함수에서 현재 선택된 날짜와 프로젝트에 유효한(진행 중인) 목표가 있는지 먼저 확인합니다.
    *   유효한 목표 필터링 조건:
        1.  현재 선택된 프로젝트(`selectedProjectId`)에 속해야 함.
        2.  상태가 'completed'(완료)가 아니어야 함.
        3.  현재 날짜가 목표 기간(`startDate` ~ `endDate`) 내에 포함되어야 함.
    *   조건을 만족하는 목표가 없으면:
        *   Bottom sheet를 띄우지 않고 `goalId`를 `undefined`(없음)로 설정하여 즉시 할 일을 저장합니다.
        *   "목표가 없는 할 일이 등록되었습니다."라는 Toast 메시지를 표시하여 사용자에게 알립니다.
    *   조건을 만족하는 목표가 하나라도 있으면:
        *   기존대로 목표 선택 Bottom sheet를 표시합니다.
